cloud-terminal/pure

The same terminal — zero backend.

This page is plain static HTML. Your browser talks to the GitHub REST API directly to start a Codespace for …, and the terminal itself is served from inside that Codespace on a privately forwarded port — guarded by GitHub's own sign-in, readable only by you.

Step 1 · Token

Kept in this tab's memory only — never stored, never sent anywhere except api.github.com. Create one under Settings → Developer settings. For private repositories the token also needs repo access.

Step 2 · Launch

Open terminal ↗

The terminal opens in a new tab on *.app.github.dev. If GitHub asks you to sign in there once, that is the private-port protection working as intended.

transport: browser → api.github.com · terminal: in-codespace bridge on a private port · no third server involved
