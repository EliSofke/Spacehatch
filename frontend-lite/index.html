spacehatch/lite

One click. No token — and no VS Code.

Both steps ride your existing GitHub session, the same mechanism VS Code Web uses: step 1 resumes or creates the codespace on GitHub's side, step 2 opens Spacehatch's own bare terminal on a privately forwarded port. Nothing to paste, nothing to configure, no third party involved.

Step 1 · Launch

Manage codespaces ↗

Opens the codespace via GitHub's quickstart page (resume-or-create), authenticated by your session. The VS Code Web tab it opens can be closed once the codespace is running — step 2 gives you the bare terminal.

Step 2 · Bare terminal (no VS Code)

Opens https://<name>-7681.app.github.dev/ — our in-codespace xterm.js bridge on a privately forwarded port, guarded by the same GitHub session (no token). Requires the target repository to carry the Spacehatch .devcontainer/ bridge. Find the name on github.com/codespaces (shown under the repo) — this page remembers it for next time, and the terminal URL itself is bookmarkable.
